# E-E-A-T / YMYL / 法令 チェックリスト(eeat-reviewer 用)

## Trust(最重要)
- [ ] PR表記: アフィリンクを含むページの冒頭に表示される状態か(prRelated と実体の一致)
- [ ] 出典: 全統計・制度記述に出典+確認日(fact-checker 成果物の表示形式確認)
- [ ] 訂正可能性: 公開日・最終更新日の両方が frontmatter にあるか
- [ ] 誇大・断定の禁止: banned-phrases.md 突合+「〜すれば必ず〜」構文の検出

## Expertise / Authoritativeness
- [ ] 執筆者・監修者が実在マスタのIDか(存在しない人物・資格の表示は最重大違反)
- [ ] 監修必須判定を**本文の実内容**で行ったか(給料・資格要件・制度・健康に触れれば必須)
- [ ] 監修者の専門と記事テーマが一致しているか(ケアマネ記事に介護福祉士単独監修等のミスマッチ検出)

## Experience
- [ ] 体験談・「現場では〜」記述が EXPERIENCE-SLOT 由来か(AIの創作体験談が紛れていないか)
- [ ] 「〜という声があります」形式の伝聞が過剰でないか(3箇所超はExperience偽装の外形)

## YMYL固有
- [ ] 医療・健康(腰痛・メンタル)に触れる場合、受診の代替になる断定をしていないか(「病院に行かなくても〜」等は reject)
- [ ] 退職・転職の判断を断定で押し付けていないか(「今すぐ辞めるべき」→ 選択肢提示の形式へ)
- [ ] 法律(労基法等)の記述が一般論の範囲か(個別ケースの法的判断は「専門機関へ」の誘導になっているか)

## 法令(日本)
- [ ] 景表法: ランキング・「おすすめ」に根拠(評価基準への言及 or editorial-policy参照)があるか
- [ ] ステマ規制: 広告であることの表示がリンクより先に目に入る構成か
- [ ] 有料職業紹介の表現: 「必ず内定」「全員年収UP」等の実績保証表現ゼロか

## AI量産外形の検出(R-03対策)
- [ ] 同型の言い回し・段落構造の機械的反復がないか
- [ ] 「いかがでしたか」「〜について解説しました」等の無内容な定型文がないか
- [ ] 各H2が固有の情報を足しているか(見出しを入れ替えても成立する記事=網羅偽装)
