## V3 の結論サマリー(3行)

1. **KB = 「出典・鮮度・IDを持つ構造化レコードの総体」**として `data/` を再定義する。記事・pSEO・ドリル・ツール・JSON-LD・内部リンクはすべてKBからの射影(成果物)であり、**公開物に書ける事実はKBに登録済みの事実のみ**(F-1原則)。
2. マスターデータは6ドメイン・約25データセット(31章)。**全レコードに共通エンベロープ(ID・出典・確認日・有効期間)を義務付け**、これがそのまま将来のRAGチャンク単位になる(実装は今しない)。
3. ai-company に**knowledge-manager / source-manager / knowledge-builder / data-validator** の4Agentを段階追加し、フローを「KB更新 → 生成 → レビュー → 公開」に組み替える(33章)。導入は V2 の P6/P8 に同期し、時間予算・ロードマップは変えない(35章)。

## 判断基準(ユーザー指定)

5〜10年の運営を見据え、**保守性・拡張性・AIとの親和性を最優先**。V2 の憲法4条(Googleアップデート耐性・著作権リスク低・AI継続運営・1人で現実的)は引き続き有効で、矛盾時は V2 憲法を優先する(データ資産化が目的化して運営が回らなくなることを禁止する — 35章 YAGNIゲート)。
