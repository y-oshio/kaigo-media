### 見送り判断の理由(記録)

- **施設DB**: 25万事業所×薄い情報はスケールドコンテンツ濫用の外形そのもの。介護サービス情報公表システムのデータ利用条件も未確認。判断基準「1人+AIで現実的」に反するため見送り。求人系DBサイトとの正面衝突も避ける。
- **用語辞典**: AI Overviews に最も食われる領域。単独ページ群にせず、記事内FAQ・ドリル解説に吸収する。

## 2. V2 の4層構造(戦略の核心)

```
第1層: データDB(pSEO)   = PVの土台。給料×都道府県×職種、加算、求人倍率、合格点推移(事実データ)
第2層: 記事500本          = PV+CVの本体。悩み・退職・夜勤・資格・施設比較・職種
第3層: 学習ドリル          = 独自性の旗艦。オリジナル問題×監修×正答率データ(誰にもコピーされない一次データ)
第4層: ツール・診断ファネル = CV装置+被リンク資産。手取り計算機・夜勤手当計算機・8タイプ診断→LINE
```

1. **「1つの主砲」から「4つの分散エンジン」へ**。V1 は過去問が70%のPVを担う一点集中型だった。V2 は最大エンジンでも全体の40%以下に抑え、コアアップデートの単一クラスタ直撃で事業が沈まない構成にする(判断基準1)。
2. **すべてのエンジンが自前データか政府統計で動く**。転載物ゼロ = 著作権リスク最小(判断基準2)。他者がコピーできない一次データ(ドリル正答率・診断集計・計算機)を蓄積し「引用される側」に回る(V1 R-09 対応の強化)。
3. **AIで回るものだけで構成**(判断基準3・4)。記事=ai-company 稼働済み。pSEO=データ+テンプレート。ドリル=AI生成+人間監修。施設DBのような人手検証が破綻する規模のものは持たない。

## 4. 差別化の再定義 — 「過去問.com に勝つ」から「誰の土俵でもない場所を取る」へ

V1 の勝ち筋は「過去問.com より高品質な解説」だった。V2 の勝ち筋は次の3点に置き換える:

1. **統計の構造化No.1**: 「介護職の給料を、公的統計ベースで、都道府県×職種×年次推移まで構造化した唯一の個人運営サイト」。競合大手(きらッコノート等)は記事はあるがDBがない。
2. **演習体験No.1**: 過去問の逐語コピー合戦(過去問.com の土俵)を降り、「監修済みオリジナル問題×弱点管理×正答率データ」という体験で差別化。実過去問がないハンデは、**試験の事実データ(合格点・パート合格制度)の速報・解説を最速で出す**ことで補う。
3. **働き手視点の比較No.1**: 施設・職種・資格の比較領域は入居者向けサイトと資格スクールの間で空白。「働く側」に一貫して張る。
