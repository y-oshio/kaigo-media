# 04. コンテンツ戦略・編集ガイドライン

> YMYL領域(転職=人生の重大決定、給料=金銭)としての品質基準と、1人運営+AIで回す制作フローを規定する。

## 1. E-E-A-T 設計

### 1-1. サイトとして満たす要件

| 要素 | 実装 |
| --- | --- |
| Experience(経験) | 記事に介護現場経験者の一次体験(監修者コメント・取材コメント)を挿入。体験の出所を明記 |
| Expertise(専門性) | 全記事に執筆者+監修者を表示。過去問解説・給料・資格記事は**有資格者(介護福祉士/ケアマネ/社会福祉士)監修を必須**とする |
| Authoritativeness(権威性) | 監修者ページ(`/supervisor/<slug>/`)に資格証明・経歴・外部実績リンク。`Person` 構造化データ + `sameAs` |
| Trust(信頼) | `/about/`(運営者の実名または法人名・所在地・連絡先)、`/editorial-policy/`(編集方針・広告方針・訂正方針)、全統計に出典+確認日、全アフィリエイトページにPR表記 |

### 1-2. 監修体制(1人運営+外部の継続パートナー)

- 監修者は**単発依頼ではなく継続パートナー2名(介護福祉士1名・ケアマネ1名)として固定契約**する。単発の無名監修者の氏名掲載は権威性シグナルとしてほぼ機能せず、「監修バッジを貼った量産アフィサイト」という既視感のある外形になるため。
- **選定条件(必須)**: 本人がWeb上で検証可能な実在エンティティであること — SNS・寄稿・登壇・他媒体執筆などの外部発信実績があり、`sameAs` で実体にリンクできる人。
- 報酬は月額の継続契約で設計(月額1〜3万円で月N本+質問対応、等)【仮説 H-20】。1ヶ月目に実見積で確定する。
- 監修の範囲を契約で明確化: 「事実誤認・危険な助言・古い制度情報の指摘」まで。文章表現は編集側の責任。過去問解説は科目適性で振り分ける。
- 監修なしで公開してよいのは職種図鑑・体験談まとめ等の低リスク記事のみ。**給料・資格要件・制度(処遇改善加算等)に触れる記事は監修必須**。
- **運営者自身のエンティティ構築も E-E-A-T 要件に含める**: 実名または一貫した屋号での発信(SNS・note等)、他媒体への寄稿・取材実績づくりを四半期タスクとして継続する(ページ上の表記だけでは権威性は積み上がらない)。

### 1-3. 広告表記(法令遵守)

- 2023年10月施行のステルスマーケティング規制(景品表示法)に基づき、アフィリエイトリンクを含む全ページの冒頭に「本ページはプロモーションが含まれています」を表示する。
- 「おすすめランキング」を作る場合は根拠(評価基準)を `/editorial-policy/` に明文化する。
- 誇大表現の禁止リスト: 「絶対」「必ず受かる」「誰でも年収○万円」等(§4 の校閲チェックに組み込む)。

## 2. コンテンツフォーマット規格

### F1: 過去問1問ページ(C1)

```
- 問題文(原文どおり。改変しない)
- 選択肢1〜5(タップで正誤表示)
- 正解
- 【オリジナル解説】← このページの独自価値。以下を必ず含む:
  - なぜその選択肢が正解か(根拠となる制度・知識)
  - 各誤答選択肢が誤りである理由(1行ずつ)
  - 覚え方・関連知識(+制度改定があった場合は「現在はこう変わっている」注記)
- 出典表記: 「第N回介護福祉士国家試験 問N(公益財団法人社会福祉振興・試験センター)」
- 関連問題リンク(同一科目3問)
```

- 解説は**過去問.com のユーザー投稿解説を上回る品質**を基準とする: 制度根拠の明示・改定情報の追記・監修者チェック。
- **定量下限(CIで機械チェック → [10章 ツール#6](./10-implementation-order-and-tools.md))**: 解説本文300字以上 / 誤答理由は各選択肢1文以上 / 根拠となる制度・法令・知識体系名を最低1つ明記。問題文・選択肢は競合と同一テキストになる以上、**解説の量と質だけがこのページの存在理由**であり、下限を割るページは公開しない(シン/スケールドコンテンツ判定対策 [R-03](./09-hypotheses-and-risks.md#r-03))。
- 制度改定で正解が変わりうる問題には「出題当時の正解」と「現行制度での注意」を併記(これが競合最大の穴)。
- 将来の差別化(Phase 3 検討): 自サイトユーザーの解答データから正答率・ひっかかりやすい誤答を集計し、競合が持たない独自データ要素としてページに追加する。

### F2: 給料統計ページ(C3)

- 数値は必ず出典(調査名・調査年・確認日)とセットで表示。「平均給与額」の定義(基本給+手当+一時金(**4〜9月支給金額の1/6**)、処遇改善加算取得事業所 — [令和6年度調査概要](https://www.mhlw.go.jp/toukei/saikin/hw/kaigo/jyujisya/24/dl/r06gaiyou.pdf) 確認日 2026-07-04)を毎ページ脚注に入れる。
- 統計の更新は年1回(処遇状況等調査は例年3〜4月頃公表)。更新時は本文の年号・数値・グラフを一括更新(半自動化 → [10章](./10-implementation-order-and-tools.md))。

### F3: 記事(C2/C4/C5/C6)

- 構成: 結論先出しリード(3行) → 目次 → H2×3〜6 → FAQ(2〜4問) → 監修者ボックス → CTA。
- 文字数は目標を置かない(クエリの必要十分で決める)。ただし競合上位3記事の網羅項目+独自価値1点(体験談/データ/診断)を必須とする。
- 1記事1クエリ群。カニバリゼーション防止のため、公開前に既存記事との対象クエリ重複をチェック(キーワード管理表 → [06章 §4](./06-data-model.md))。

## 3. AI活用制作フロー(1人運営の中核)

> AIはドラフト生成器ではなく「構成〜検証の全工程の副操縦士」として使う。ただし**公開判断・一次情報確認・体験談の真正性は必ず人間**が担う。スケールドコンテンツ濫用([R-03](./09-hypotheses-and-risks.md#r-03))を避けるため、「AIが書いた記事を無検証で量産する」ことを明示的に禁止する。

### 記事制作パイプライン(1本あたり実働60〜90分【仮説 H-08】)

| 工程 | 担当 | 内容 |
| --- | --- | --- |
| 1. キーワード選定 | 人間(月次でまとめて) | 優先順位マトリクス([02章 §3](./02-keyword-strategy.md))から当月分を選定 |
| 2. SERP調査+構成案 | AI+人間確認(10分) | 上位10記事の網羅項目・検索意図をAIが分析、見出し案を生成。人間が独自価値ポイントを1つ決める |
| 3. ドラフト生成 | AI(5分) | 編集ガイドライン(本章)をプロンプトに含めて生成。出典が必要な箇所は `【要出典】` プレースホルダで出力させる |
| 4. ファクトチェック | 人間+AI(20〜30分) | `【要出典】` を一次情報(厚労省・試験センター・e-Stat)で全数確認しURL+確認日を記録。AIには「この主張の一次出典を探せ」だけをやらせ、真偽判断は人間 |
| 5. 独自価値の注入 | 人間(15分) | 体験談・監修者コメント・自サイトデータ・診断への文脈接続 |
| 6. 校閲 | AI+人間(10分) | 禁止表現・PR表記・内部リンク規則([03章 §3](./03-site-architecture.md))・表記ゆれの機械チェック |
| 7. 監修(対象記事のみ) | 外部監修者(非同期) | 指摘反映後に公開。監修済みバッジと監修日を表示 |

### 過去問解説パイプライン(1回分125問を2〜3週間【仮説 H-09】)

1. 試験センター公表のPDF(問題・正答)を構造化データへ変換(自動化ツール #1 → [10章](./10-implementation-order-and-tools.md))
2. AIが解説ドラフトを一括生成(根拠制度を必ず引用させる)
3. 人間が全問の正解番号を公式正答表と突合(**ここは絶対に省略しない**)
4. 監修者が科目単位でレビュー(誤答理由の妥当性・制度改定の見落とし)
5. 公開はまとめてではなく科目単位で逐次。**公開ゲート**: 直前公開分のインデックス登録率が90%を下回ったら次の公開を停止し、品質監査(解説の定量下限・重複判定)を先に行う

## 4. 品質管理・運用ルール

- **公開前チェックリスト**(全ページ共通): 出典+確認日 / PR表記 / 監修要否判定 / 内部リンク1本以上受け / 構造化データ検証 / タイトル・メタ(クエリ含有+クリック動機)。
- **更新ポリシー**: 制度改定(介護報酬改定は3年周期、処遇改善は毎年度)・統計公表・試験実施のカレンダーを持ち、影響ページを更新する(更新カレンダーは [06章 §4](./06-data-model.md) のデータで管理)。記事には「公開日」と「最終更新日」を両方表示。
- **リライト基準**: 公開3ヶ月後にGSCで順位11〜30位のページを抽出し、月4〜8本をリライト(新規制作より優先度を上げる時期の判断は [08章](./08-kpi-roadmap.md))。
- **訂正ポリシー**: 誤りの指摘を受けた場合は48時間以内に確認・訂正し、訂正履歴をページ内に残す(`/editorial-policy/` に明記)。
