## 監修完了後の処理(人間+publish-manager)

1. 指摘を fact-checker(事実系)/ writer(表現系)に振り分けて反映
2. frontmatter の `supervisorId` と `sources` を確定
3. 監修者ページ(/supervisor/{slug}/)の監修記事一覧に追加
4. manifest の status を awaiting-supervisor → ready に更新(publish-manager を再実行)
