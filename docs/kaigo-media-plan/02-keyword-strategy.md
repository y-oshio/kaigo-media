# 02. キーワード戦略

> 検索ボリュームの数値は **すべて【仮説】** であり、実測(Google キーワードプランナー / ahrefs / GSC)による検証が必要。検証手順は §5、仮説IDは [09. 仮説・リスク台帳](./09-hypotheses-and-risks.md) に登録済み。

## 1. 戦略の基本方針

1. **求人トランザクショナル(「介護 求人 ○○市」)は捨てる** — 実求人データを持つサイトのみが上位表示される領域([01章 §2-1](./01-market-analysis.md))。
2. **「ページ数がスケールするクエリ群」を主軸にする** — 過去問(1問1クエリ)・給料統計(都道府県×職種)は、テンプレート×データでページを量産でき、1人運営でも網羅できる。
3. **CVに近いクエリは本数で稼がず1本の質で獲る** — 「介護職 辞めたい」等の悩み系は記事数が少なくても1位を取れば大きい。
4. **AI Overviews 耐性で優先度を補正する** — 「〜とは」系の単純情報クエリは優先度を下げ、演習・データ・診断などページ上での行動が発生するクエリを優先する。

### C1: 過去問・試験対策(PVの主砲)

- 対象試験: **介護福祉士国家試験**(第38回=2026年1月実施まで公表済み)→ **ケアマネ試験**(介護支援専門員実務研修受講試験)の順に展開。初任者研修は修了試験が非公開のためオリジナル練習問題で対応。
- クエリ例と仮説ボリューム(月間、通常期):
  - 「介護福祉士 過去問」: 3〜6万【仮説 H-01】。12〜1月に2〜4倍化【仮説 H-01】
  - 「介護福祉士 過去問 第37回」「介護福祉士 合格点」「介護福祉士 解答速報」: 各数千〜数万(試験期集中)【仮説 H-01】
  - **パート合格制度(第38回〜)の新規需要**: 「介護福祉士 パート合格 とは」「介護福祉士 Bパート 合格点」「パート合格 繰り越し」等。旧制度前提の競合記事が多く残る移行期は先行獲得のチャンス【仮説 H-01】
  - 個々の問題文の一部・科目名検索(「こころとからだのしくみ 過去問」等): ロングテール多数【仮説 H-01】。パート別再受験者の増加で科目別・パート別演習クエリの比重が上がる見込み
- ページ設計: 1問1ページ(問題→選択肢→解答→**オリジナル解説**→関連問題)+ 回次別一覧 + 科目別一覧。詳細は [03章](./03-site-architecture.md)。
- ページ数見込み: 介護福祉士 125問 × 直近8回 = 1,000ページ、ケアマネ 60問 × 8回 = 480ページ。
- **前提条件**: 試験問題の転載許諾の確認([R-01](./09-hypotheses-and-risks.md#r-01))。不許諾の場合は「オリジナル一問一答」形式へ転換するプランBを [09章](./09-hypotheses-and-risks.md) に定義。

### C2: 資格・研修

- クエリ例: 「初任者研修 費用」「初任者研修 働きながら」「実務者研修 初任者研修 違い」「介護福祉士 受験資格 実務経験」「喀痰吸引研修 とは」: 各1千〜1万【仮説 H-13】
- CV先: かいご畑(資格取得支援つき派遣)等、資格系案件と相性が良い([07章](./07-conversion-design.md))。

### C3: 給料・年収

- クエリ例: 「介護福祉士 給料」1〜3万、「介護 夜勤 手当 相場」「介護職 手取り」「処遇改善加算 いくら」: 各1千〜1万【仮説 H-14】
- データ駆動ページ: 「【2026年版】○○県の介護職の給料」47ページ +「職種・資格別給料」8〜12ページ。出典: 令和6年度介護従事者処遇状況等調査・賃金構造基本統計調査([06章 §3](./06-data-model.md))。
- ファクト: 離職理由の1位は「職場の人間関係」34.3%、「収入が少なかった」は4位16.6%(令和5年度介護労働実態調査 [介護労働安定センター](https://www.kaigo-center.or.jp/content/files/report/2023_jittai_chousagaiyou.pdf) 確認日 2026-07-04)。給料クラスタをCVに強く接続する根拠は「離職動機の大きさ」ではなく、**給料クエリは比較・行動に近い検索意図を持つ**こと(意図ベースの根拠)。人間関係起点のC4はさらに太い動機源として扱う。

### C4: 悩み・退職・転職(CVの主戦場)

- クエリ例: 「介護職 辞めたい」1〜3万【仮説 H-15】、「介護 人間関係 最悪」「介護職 向いてない」「介護 転職 タイミング」「介護 転職サイト おすすめ」: 各1千〜1万【仮説 H-15】
- 「介護 転職サイト おすすめ」はアフィリエイト激戦区。新規ドメインでの短期上位化は困難【仮説 H-16】→ 12ヶ月以降の挑戦枠とし、序盤は「辞めたい」「向いてない」等の悩み起点クエリ×診断誘導で独自ポジションを取る。

### C5: 施設種別

- クエリ例: 「特養 老健 違い 働く側」「デイサービス 仕事内容 きつい」「有料老人ホーム 介護職 給料」: 各数百〜数千【仮説 H-17】
- 注意: 入居者・家族向け検索と混在するクエリが多い。**タイトル・導入で「働く人向け」を明示**し意図を分離する(既存診断の8タイプ = tokuyo / rouken / dayservice / homecare / paidhome / nightshift / beginner / caremanager と1:1で接続できる)。

### C6: 職種図鑑

- クエリ例: 「生活相談員 なるには」「サービス提供責任者 仕事内容」「ケアマネ 仕事内容」: 各数百〜数千【仮説 H-18】
- キャリアパス記事として内部リンクのハブになり、C1(資格)・C3(給料)へ回遊させる。

### C7: 診断・ツール

- 検索クエリ: 「介護 適性 診断」「介護 施設 どこが向いてる」: 小規模【仮説 H-19】。ただし主用途は**検索流入の受け皿ではなくサイト内ファネル**(C4/C5からの誘導先)。

## 3. 優先順位マトリクス

評価軸: PVポテンシャル / CV近接度 / 競合強度(低いほど良い) / 制作コスト(低いほど良い)。

優先度は [08章のロードマップ](./08-kpi-roadmap.md)(0〜3 / 3〜6 / 6〜12 / 12ヶ月〜)と1:1対応させる。

| 優先度 | クラスタ | 理由 |
| --- | --- | --- |
| P0(0〜3ヶ月) | C2 資格・C5 施設種別・C4 悩み(計30本前後) | 競合中程度・新規ドメインでも入りやすいロングテールから着手し、E-E-A-T の土台を作る |
| P0(0〜3ヶ月) | C1 過去問ハブ・勉強法記事(許諾不要分)を先行公開 + 許諾取得後に直近2〜3回分の逐次公開開始 | 2027年1月の試験期スパイクに評価が間に合うよう、クラスタの評価を最も早く温め始める |
| P1(3〜6ヶ月) | C1 累計5回分(625問)+ C3 給料記事・職種別統計 | 試験期(12〜1月)を5回分+速報系コンテンツで迎える。給料記事でCV導線を開通 |
| P2(6〜12ヶ月) | C1 8回分完成+第39回即日投入 + C3 給料統計DB(47都道府県)+ C6 職種図鑑 | データ駆動ページの投入。統計は毎年更新で複利化 |
| P3(12ヶ月〜) | C1 ケアマネ(2027年10月試験に照準)+ C4「転職サイト おすすめ」等の激戦クエリ + リライト | ドメイン評価が育ってから挑戦 |

## 4. AI Overviews / ゼロクリック補正

| クエリ型 | AI Overviews 代替リスク | 対応 |
| --- | --- | --- |
| 「〜とは」定義型 | 高 | 単独記事にせず、ハブ記事の一部+FAQとして扱う |
| 過去問演習 | 低(演習行動はAIで代替されにくい) | 主砲として投資 |
| 統計・地域データ | 中(要出典データはAIが引用側に回る) | 一次データ加工+グラフで「引用元」ポジションを狙う |
| 悩み相談型 | 中 | 体験談・監修者コメントで一次情報性を担保 |

## 5. 仮説検証の手順(ローンチ前・運用中)

1. **ローンチ前(0ヶ月目)**: Google キーワードプランナー(要広告アカウント)で §2 の主要50クエリの月間ボリュームを実測し、本書の仮説値を上書きする。可能なら ahrefs/Ubersuggest の7日トライアルで競合の推定流入も取得。
2. **運用中(毎月)**: GSC のクエリレポートで「表示回数はあるがCTR/順位が低いクエリ」を抽出し、リライト・新規記事の優先度を再計算する(自動化ツール案: [10章 §2](./10-implementation-order-and-tools.md))。
3. **試験期(12〜2月)**: 過去問クラスタの実測スパイク倍率を記録し、翌年のPVモデル([08章](./08-kpi-roadmap.md))を補正する。
