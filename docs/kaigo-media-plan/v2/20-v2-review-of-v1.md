## 2. 影響の定量評価

### 2-1. PVへの影響

- **失うもの**: V1 試験期モデルの C1 = 30〜70万PV【旧H-02】のうち、(a)「第N回 問N」等の指名・逐語検索、(b)「介護福祉士 過去問」ヘッドクエリの本体。これらは実物の過去問がないと原理的に取れない。
- **残るもの・代替できるもの**: 学習需要そのもの(一問一答・予想問題・科目別演習・勉強法)、試験の**事実データ**(日程・合格点・合格率は著作物ではない)による「合格点」「解答速報まとめ」「日程」クエリ、パート合格制度解説。
- 正味影響(仮説): 試験期PVで **−40〜55万**。代替エンジン(ドリル+統計pSEO+記事増)で **+30〜50万** を24〜36ヶ月かけて積む(→ [22章](./22-v2-traffic-model.md))。

### 2-2. CVへの影響(結論: 軽微〜中立)

- V1 の収益式は最初から**過去問PVを除外**していた(基数=非過去問30万PV。07章 §4)。過去問の収益貢献は「LINE登録→3月配信」の間接経路のみ。
- V2 では CV対象PV(記事・給料・資格・ツール)がむしろ主役になり、**同じ100万PVならCV基数は30万→60〜90万に増える**。収益中央値シナリオは維持〜上方(22章 §4)。
- 失うのは LINE 登録の主要供給源(旧H-11)。ドリル+「毎日3問配信」訴求で部分代替(26章 §6)。

### 2-3. 開発工数への影響(結論: ほぼ中立)

| 項目 | V1 | V2 |
| --- | --- | --- |
| ツール#1 過去問PDFパイプライン | 16〜24h | **削除**(PDFレイアウト解析が最大の不確実要素だった) |
| 過去問エンジン(#9) | 16h | ドリルエンジンに転用 12〜16h(テンプレ・SSG・データ構造は流用) |
| ドリル生成パイプライン | — | **新設** 8〜12h(PDF解析がない分、#1より軽い) |

| 合計 | 約110h | **約100〜110h(同等)** |

### 2-4. 著作権リスクへの影響(結論: 大幅に低下、ただし新リスク2件)

- **消えるリスク**: R-01(転載許諾)・R-07(ケアマネ権利帰属)が事業ブロッカーでなくなる。第三者著作物(図表)の個別権利処理・欠番運用も不要。
- **新たに生じるリスク**(→ 22章 §5 で台帳化):
  - **R-10 翻案リスク**: オリジナル問題が実在の過去問に酷似すると複製・翻案侵害になりうる。対策 = クリーンルーム生成(過去問本文をAI入力に使わない。出題基準・法令・公的テキスト概念のみから生成)+人間監修者による市販問題集との目視突合(26章 §5)。
  - **R-11 事実データの範囲逸脱**: 合格点・合格率・日程・出題基準の項目名は事実(著作権なし)として使えるが、**正答表の丸ごと転載や出題基準の表現の逐語コピーはしない**。事実の記述に留める。
- 残る一般リスク: 統計は政府標準利用規約(出典明記で利用可)準拠 — V1 と同じ。

## 3. 結論 — 何を捨て、何を残すか

| 資産 | 判定 |
| --- | --- |
| 過去問1問ページ(逐語転載) | **捨てる**(許諾が出た場合のみオプション実装。URL `/kakomon/` とスキーマ origin:'kakomon' は凍結保存) |
| 試験の事実データ(ExamRound: 日程・合格点・合格率) | **残す**(著作物でない。「合格点推移」「速報まとめ」はV2でも主力) |
| 科目マスタ・出題領域の分類 | **残す**(ドリルの論点マップの骨格になる) |
| プランB(オリジナル一問一答・origin:'original'・drills/) | **昇格**: 非常用→主力。V1 の C-2 指摘対応で設計済みだったため、情報設計のやり直しは発生しない |
| 給料統計DB・記事クラスタ・診断ファネル | **残す+拡大**(V2 の主役。→ 21章) |
| ai-company 記事パイプライン | **ほぼ無傷**(C1 を最初から対象外にしていた。→ 24章) |
